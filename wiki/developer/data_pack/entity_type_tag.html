Entity type tag ​

An entity type tag is a group of entity types. It can be used in type target selector arguments and loot table conditions with #<resource location>, which checks if the entity's type matches any of the entity types specified in the tag. Entity type tags are also used to control a number of other gameplay features; see below for the use of each tag.

Data pack or mod developers who may wish to add a compatibility layer between their data pack or mod and Seven Elements may do so by adding their entity type into the proper Seven Elements entity type tag.

List of tags ​

deals_pyro_damage ​

Contains entity types that innately deal 1U Pyro with the tag seven-elements:mob_damage if no previous elemental infusion exists or the damage being dealt is Physical.

  • #seven-elements:deals_pyro_damage (2 values)
    • minecraft:blaze
    • minecraft:magma_cube

deals_hydro_damage ​

Contains entity types that innately deal 1U Hydro with the tag seven-elements:mob_damage if no previous elemental infusion exists or the damage being dealt is Physical.

  • #seven-elements:deals_hydro_damage (4 values)
    • minecraft:axolotl
    • minecraft:elder_guardian
    • minecraft:drowned
    • minecraft:guardian

deals_anemo_damage ​

Contains entity types that innately deal 1U Anemo with the tag seven-elements:mob_damage if no previous elemental infusion exists or the damage being dealt is Physical.

  • #seven-elements:deals_anemo_damage (no values)

  • #seven-elements:deals_electro_damage (no values)

  • #seven-elements:deals_geo_damage (no values)

ignored_targets ​

Contains entity types that are ignored as targets for both the Hyperbloom Elemental Reaction and the Elemental Shard.

Please consider only using this tag for entities that shouldn't be valid targets of these mechanics.

  • #seven-elements:ignored_targets (3 values)
    • minecraft:armor_stand
    • seven-elements:crystallize_shard
    • seven-elements:dendro_core
